# Project Context: Tooling Select System

## 1. Project Overview & Main Target
**ชื่อโปรเจค:** Tooling Select
**เป้าหมายหลัก (Main Target):** ระบบแอปพลิเคชันสำหรับประเมินและตัดสินใจเลือกใช้ "เครื่องมือ (Tooling No.)" ที่มีอยู่ในสต็อก (Stock) ให้เหมาะสมกับสเปค (Spec) และประเภทของงานตามหมายเลขควบคุม (CN - Control Number) แต่ละรายการ ระบบนี้จะช่วยวิเคราะห์ว่า Tooling ที่มีอยู่สามารถใช้งานได้หรือไม่ หรือจำเป็นต้องดำเนินการ **"ออกแบบ Tooling ใหม่ (New Design)"**

## 2. Core Constraints & System Architecture
**ข้อจำกัดสำคัญที่สุด (Critical Rule):**
* **Dynamic Configuration:** ระบบต้องถูกออกแบบให้เป็น **Data-Driven** อย่างสมบูรณ์ 
* **ห้ามแก้ไขโค้ด (No Code Modification):** เมื่อมีการเพิ่ม Machine ใหม่, Tooling ใหม่ หรือ Formula (สูตรการคำนวณ) ใหม่ ระบบจะต้องรองรับการเพิ่มข้อมูลเหล่านี้ผ่าน UI หรือ Database โดยตรงเท่านั้น ห้ามมีการ Hardcode ตัวแปร เครื่องจักร หรือสูตรคำนวณลงใน Source Code เด็ดขาด

## 3. Application Modules & Flow
ระบบประกอบด้วยหน้าต่างและโมดูลการทำงานหลักดังต่อไปนี้:

### 3.1. Main Page (หน้าหลักสำหรับการค้นหา)
* **Flow:** ผู้ใช้กรอกหมายเลข `CN` ในช่องค้นหา -> กดค้นหา -> ระบบแสดงผลลัพธ์
* **Display format:** แสดงผลลัพธ์แบบจัดกลุ่ม (Hierarchical Tree) ดังนี้:
    * **Machine** (เครื่องจักร)
        * **Tool Name** (ประเภท/ชื่อเครื่องมือ)
            * **Tool No.1** (เครื่องมือชิ้นที่ 1 ในสต็อกที่เข้าเงื่อนไข)
            * **Tool No.2** (เครื่องมือชิ้นที่ 2 ในสต็อกที่เข้าเงื่อนไข)
* **Alert & Notification:** ที่ระดับ `Machine` ต้องมีการแจ้งเตือน (Badge/Text) แสดงสถานะว่า *"ค้นพบ Tool No. กี่ชิ้น จากที่ต้องใช้ทั้งหมดกี่ชิ้น"* เพื่อให้ผู้ใช้ทราบทันทีว่าชิ้นส่วนใดขาดและต้องเข้ากระบวนการ New Design

### 3.2. Add Tool (การจัดการข้อมูลเครื่องจักรและประเภทเครื่องมือ)
* แสดงตารางรายชื่อ `Machine` และ `Tool Name` ที่ใช้งานร่วมกับเครื่องจักรนั้นๆ
* มีฟังก์ชันให้ผู้ใช้สามารถเพิ่ม/ลบ/แก้ไข `Machine` และกำหนด `Tool Name` ย่อยของแต่ละเครื่องจักรได้แบบ Dynamic

### 3.3. Tool List (คลังเครื่องมือที่มีในสต็อก)
* แสดงรายการ `Tool No.` ทั้งหมดที่มีอยู่ในระบบ
* **Filter System:** 1.  Dropdown ที่ 1: เลือก `Machine`
    2.  Dropdown ที่ 2: เลือก `Tool Name` (รายการจะแปรผันตาม Machine ที่เลือกในข้อ 1)
    3.  เมื่อเลือกครบ ระบบจะแสดงตารางรายการ Tool No. ที่ตรงกับเงื่อนไข

### 3.4. Part Spec (ข้อมูลสเปคชิ้นงาน)
* แสดงรายการหมายเลข `CN` และรายละเอียด `Spec Part` ทั้งหมดของ CN นั้นๆ
* มีฟังก์ชันเพิ่ม (Add), แก้ไข (Edit) และลบข้อมูลได้

### 3.5. Formula (ระบบสูตรการคำนวณและตัวแปรสำหรับ Tooling)
* **Logic:** ในแต่ละ `Tool Name` ของแต่ละ `Machine` จะมีชุดตัวแปร (Variables) เฉพาะตัว 
* **Variable Source:** ค่าของตัวแปรสามารถดึงมาจาก:
    1.  ค่าโดยตรงจาก `Part Spec` (เช่น `dim_a = od`, `dim_b = w`)
    2.  ค่าจากการคำนวณทางคณิตศาสตร์ (เช่น `dim_c = (a/b) * 0.8`)
* **New Design Workflow:** หากค้นหาในสต็อกแล้วไม่พบ Tooling ที่เข้าเงื่อนไข ระบบจะนำค่าตัวแปรที่ผ่านการคำนวณตาม Formula นี้ ไปใช้เป็นพารามิเตอร์เริ่มต้นสำหรับสร้าง Tool ใหม่ (เช่น ส่งออกค่า `dim_a`, `dim_b`, `dim_c` เพื่อนำไปอัปเดตโมเดลในโปรแกรม CAD ได้ง่าย)
* **UX/UI Requirement:** หน้าจอ UI สำหรับตั้งค่า Formula ต้อง **"เข้าใจง่าย"** และ **"ใช้งานง่าย"** ผู้ใช้ (ที่อาจไม่ใช่โปรแกรมเมอร์) ต้องสามารถเลือกตัวแปรจาก Part spec, กำหนดค่าตัวแปรใหม่ และเขียนสูตรคณิตศาสตร์ผูกเงื่อนไขได้อย่างสะดวก (เช่น การใช้ Drag & Drop, Select box หรือ Expression Builder ที่มีตัวอย่างให้ดู)

### 3.6. Selection Rule (กฎการคัดเลือก)
* ตารางกำหนดเงื่อนไข (Conditions/Rules) ในการยอมรับ (Acceptance) ว่า Tooling No. ในสต็อกชิ้นนั้นๆ สามารถนำมาใช้กับ Part Spec ปัจจุบันได้หรือไม่ (เช่น ค่าพิกัดความเผื่อ (Tolerance), ระยะ Clearance หรือ Material type)

### 3.7. Machine Limit (ข้อจำกัดของเครื่องจักร)
* ตารางกำหนดขีดความสามารถ (Capacity) ของเครื่องจักรแต่ละตัว (เช่น ขนาดชิ้นงานสูงสุด-ต่ำสุด (Max/Min Workpiece Size), น้ำหนักสูงสุดที่รองรับได้) เพื่อกรองว่างาน CN นี้สามารถนำไปผลิตบนเครื่องจักรใดได้บ้าง
