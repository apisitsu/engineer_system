====================================================
  ตารางฐานข้อมูลที่ใช้ใน Tooling Select
====================================================

── Static Fixtures (fixtureLogic.js) ──────────────
ตารางเหล่านี้ถูก query โดยตรงทุกครั้งที่ค้นหา C/N

1. spec_process          SELECT  (ดึงข้อมูล Part Spec จาก C/N)
2. tooling_ksb22g        SELECT  (JAW + BACK PLATE สำหรับ KS-B22G)
3. tooling_ksb80         SELECT  (JAW + BACK PLATE สำหรับ KS-B80)
4. tooling_tsg300        SELECT  (CHUTE COVER + CARRIER สำหรับ TSG-300)
5. tooling_ks03a         SELECT  (ทุก Tool type สำหรับ KS-03A / KS-B22RD)
6. tooling_ks400b        SELECT  (WORK DRIVER, LOADING CHUTE, SUPPORT BLOCK, PLUG A/B)
7. tooling_ks500rd       SELECT  (LOADING PINTLE + WORK DRIVER สำหรับ KS500RD)
8. tooling_ks400b5       SELECT  (ทุก Tool type สำหรับ KS400B5)
9. tooling_ks400b6       SELECT  (ทุก Tool type สำหรับ KS400B6)

── Dynamic Rules (dynamicLogic.js) ────────────────
ใช้สำหรับเครื่องจักรที่กำหนดกฎผ่าน UI (ไม่ hardcode)

10. mtc_selection_rules  SELECT  (ดึงกฎทั้งหมดที่ is_active = true)
11. tooling_*            SELECT  (ตารางที่ระบุใน mtc_selection_rules.target_tool_table
                                  — dynamic ขึ้นอยู่กับข้อมูลในตาราง rules)

── Inventory CRUD (tooling_select.js) ─────────────
ใช้สำหรับจัดการข้อมูลใน Tool List UI

12. tooling_*            SELECT / INSERT / UPDATE / DELETE
                         (ตารางชื่อ tooling_* ทุกตาราง ยกเว้น tooling_inspect)

── System (information_schema) ────────────────────
13. information_schema.tables    SELECT  (ตรวจสอบว่า table มีอยู่จริง)
14. information_schema.columns   SELECT  (ดึง column list เพื่อป้องกัน SQL injection)
