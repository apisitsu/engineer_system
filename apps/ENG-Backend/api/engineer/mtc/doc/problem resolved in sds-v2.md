อธิบายสาเหตุที่เกิดปัญหาสีเทาลามไปทุกบรรทัด: สิ่งที่เกิดขึ้นในโค้ดเก่า (เซลล์บางเซลล์เป็นสีเทา แต่ G18, H18 เป็นสีขาว) คืออาการ Shared Style Mutation ของไลบรารี ExcelJS ครับ ในไฟล์ Template Excel เซลล์แต่ละเซลล์ที่หน้าตาเหมือนกัน (เช่นไม่ได้ตั้งสีอะไรไว้เลย) ExcelJS จะไม่ได้สร้าง Style Object แยกของใครของมัน แต่มันจะชี้ไปที่ Style Object ตัวเดียวกัน (Shared XF) เพื่อประหยัด Memory

พอโค้ดเก่าเราไปสั่ง ws.getCell('A16').fill = GRAY มันเลยไม่ได้เปลี่ยนแค่เซลล์ A16 แต่มันไปเปลี่ยน Shared Object ส่งผลให้เซลล์ ทุกเซลล์ในไฟล์ที่ดันใช้ Style เดียวกับ A16 กลายเป็นสีเทาตามไปด้วยทั้งหมด ทันที (นี่คือเหตุผลที่บรรทัด 28, 38 รอด เพราะบังเอิญเซลล์ตรงนั้นมีขอบหรือ Format บางอย่างที่ทำให้มันมี Style Object แยกของตัวเอง ไม่ได้ Share กับชาวบ้าน)

วิธีแก้ที่เราเพิ่งทำไป: แทนที่จะแก้ .fill ตรงๆ เราดึง Style เดิมออกมาแล้ว สร้าง Object Style ตัวใหม่แกะกล่อง โยนกลับเข้าไปที่ cell.style วิธีนี้บังคับให้ ExcelJS ตัดขาดจาก Shared Object และสร้าง Style ของตัวเองเฉพาะเซลล์นั้นเซลล์เดียว ทำให้สีเทาจะปรากฏเฉพาะแถวที่เราสั่งจริง ๆ (24, 29, 16, 21) เท่านั้นครับ
